# RIBRE 運用ガードレール

Last updated: 2026-06-02

## 目的

売上・仕入・証憑データを壊しにくくするための最低限の運用ルールです。

## データを書き換える前

- 月締め、同期 Pull、復元、CSV 大量取込の前にバックアップを確認する。
- 自動スナップショットは `localStorage` の `ribre_auto_snapshots_v1` に最大 10 世代保存される。
- 作業前に不安がある場合は、バックアップ画面から JSON をダウンロードする。

## 表示と入力

- CSV、OCR、手入力、クラウド同期の値は、画面表示時に HTML として解釈しない。
- 商品名・メモには業務上必要な文字だけを入れる。HTML タグやスクリプト断片は入力しない。

## 日付

- 画面の「今日」は日本時間の年月日を使う。
- 日次レポートと手入力の既定日付が、朝の時間帯に前日にならないことを確認する。

## 証憑

- localStorage には大きな画像本体を残さない方針。
- 長期保存が必要な証憑は Supabase Storage などに保存し、URL を証憑レコードに残す。

## 同期

- Pull はローカルの売上・仕入を上書きする操作。
- Pull 前には自動スナップショットが作成されるが、重要作業中は手動バックアップも取る。
- 複数端末で同時に入力する場合は、片方で Push してからもう片方で Pull する。

## 確認項目

- Console に `Uncaught` / `TypeError` が出ていない。
- 売上一覧、仕入一覧、同期ログで入力文字がそのまま文字として表示される。
- 同期 Pull 後に `ribre_auto_snapshots_v1` が作成されている。
